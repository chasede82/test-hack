## 1. 사이트맵

```
MeetSync
│
├── [인증]
│   ├── 로그인
│   └── 회원가입
│
├── [홈 / 대시보드]                          ← 로그인 후 진입점
│   ├── 내 할 일 (My Todos)
│   │   ├── 진행 중인 투두 목록
│   │   └── 완료된 투두 목록
│   └── 최근 회의록 목록
│
├── [채널]
│   ├── 채널 목록 (사이드바)
│   └── 채널 상세
│       ├── 채팅 메시지 피드
│       │   └── 회의록 카드 (인라인)
│       └── 녹음 입력 패널
│           ├── 파일 업로드 (F-01)
│           └── 인앱 실시간 녹음 (F-08)
│
└── [회의록]
    ├── 회의록 목록
    └── 회의록 상세
        ├── 요약
        ├── 주요 논의 사항
        ├── 결정 사항
        └── 참여자별 투두리스트
```

## 2. 글로벌 내비게이션

전체 앱에 공통으로 적용되는 2단 레이아웃입니다.

```
┌──────────────────────────────────────────────────────────┐
│  [로고]   채널명                        [프로필 아이콘]   │  ← 상단 헤더
├──────┬───────────────────────────────────────────────────┤
│      │                                                   │
│ 사   │                                                   │
│ 이   │           메인 콘텐츠 영역                         │
│ 드   │                                                   │
│ 바   │                                                   │
│      │                                                   │
└──────┴───────────────────────────────────────────────────┘
```

### 사이드바 구성 요소

| 요소 | 설명 | 연결 화면 |
|---|---|---|
| 🏠 홈 (대시보드) | 내 투두 + 최근 회의록 | 홈 대시보드 |
| 💬 채널 목록 | 소속된 채널 리스트 | 채널 상세 |
| + 새 채널 | 채널 생성 버튼 | (모달) |
| 📋 회의록 | 전체 회의록 목록 | 회의록 목록 |

## 3. 화면 정의

### 3.1 홈 대시보드

**진입 경로**: 로그인 완료 후 자동 이동 / 사이드바 홈 아이콘
**연관 기능**: F-04, F-05

```
┌─────────────────────────────────────────────────────────┐
│  안녕하세요, [이름]님 👋                                  │
├─────────────────────────┬───────────────────────────────┤
│  📋 내 할 일             │  📁 최근 회의록                │
│                         │                               │
│  [ ] 기획안 초안 작성    │  · 3/24 주간 팀 미팅          │
│      마감: 3/26 · 주간팀 │  · 3/22 신규 기능 리뷰        │
│                         │  · 3/20 디자인 싱크            │
│  [ ] API 명세 검토       │                               │
│      마감: 3/25 · 기능리뷰│                              │
│                         │                               │
│  [완료된 항목 보기 ▾]    │  [전체 보기 →]                │
└─────────────────────────┴───────────────────────────────┘
```

**콘텐츠 요소**

| 영역 | 표시 정보 | 정렬 기준 |
|---|---|---|
| 내 할 일 — 진행 중 | 투두 내용, 마감일, 출처 회의명 | 마감일 오름차순 |
| 내 할 일 — 완료 | 완료된 투두 (접기/펼치기) | 완료 처리 일시 내림차순 |
| 최근 회의록 | 회의명, 날짜 (최대 5건) | 생성일 내림차순 |

**인터랙션**

- 투두 체크박스 클릭 → 완료 처리, "완료된 항목"으로 이동 (F-05)
- 투두 행 클릭 → 마감일 인라인 편집 가능 (F-05)
- 회의록 항목 클릭 → 회의록 상세 이동

### 3.2 채널 상세

**진입 경로**: 사이드바 채널 클릭
**연관 기능**: F-01, F-06, F-07, F-08

```
┌─────────────────────────────────────────────────────────┐
│  # 주간-팀-미팅                          [🎙️ 녹음] [📁 업로드] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ─────────────────  2026-03-24  ──────────────────      │
│                                                         │
│  [챈들러]  오늘 회의 시작합니다!               오전 10:00 │
│                                                         │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 🤖 AI 회의록이 생성되었습니다                     │   │
│  │ 📄 2026-03-24 주간 팀 미팅                        │   │
│  │ 요약: 신규 기능 개발 일정 조율 및 역할 분담 확인   │   │
│  │ 투두 3건 생성됨                    [회의록 보기 →] │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  [챈들러]  확인했습니다 감사해요!              오전 10:07 │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  메시지를 입력하세요...                    [전송 ↵]      │
└─────────────────────────────────────────────────────────┘
```

**콘텐츠 요소**

| 요소 | 설명 |
|---|---|
| 채널명 헤더 | 현재 채널 이름 |
| 🎙️ 녹음 버튼 | 인앱 실시간 녹음 패널 열기 (F-08) |
| 📁 업로드 버튼 | 파일 업로드 다이얼로그 열기 (F-01) |
| 채팅 피드 | 시간순 메시지 목록, 날짜 구분선 (F-06) |
| 회의록 카드 (인라인) | AI 회의록 자동 게시 카드 (F-07) |
| 메시지 입력창 | 텍스트 + 이모지 입력 (F-06) |

**AI 처리 상태 표시 (F-02)**

| 상태 | 채널 내 표시 |
|---|---|
| 업로드 중 | "📤 녹음 파일을 업로드하는 중입니다... (67%)" |
| 분석 중 | "🤖 AI가 회의록을 생성 중입니다..." |
| 완료 | 회의록 카드 자동 게시 |
| 오류 | "⚠️ 회의록 생성에 실패했습니다. 다시 시도해 주세요." |

### 3.3 녹음 입력 패널 (슬라이드업 / 모달)

**진입 경로**: 채널 상세 → 🎙️ 녹음 버튼
**연관 기능**: F-08

```
┌─────────────────────────────────────────────────────────┐
│  🎙️ 회의 녹음                                    [✕ 닫기] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              ●  00:04:32                                │
│           [  ■ 정지  ]   [  ⏸ 일시정지  ]               │
│                                                         │
│  마이크: 내장 마이크 (MacBook Air)              [변경]   │
│                                                         │
│  ─────── 또는 ───────                                   │
│                                                         │
│  [ 📁 녹음 파일 업로드 ]                                 │
│  mp3, mp4, wav, m4a · 최대 500MB                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**상태 전이**

```
[대기]  →  [녹음 중]  →  [저장 완료 / AI 처리 시작]
  ↑             ↓
  └── [일시정지] ←→ [재개]
```

**인터랙션**

- 🎙️ 버튼 클릭 → 브라우저 마이크 권한 요청 (미허용 시 안내 모달)
- 녹음 중 경과 시간 실시간 표시
- 정지 → 자동으로 STT + 회의록 생성 파이프라인 실행
- 파일 업로드 탭 전환 가능 (F-01)

## 4. 핵심 사용자 플로우

### Flow A — 파일 업로드 → 회의록 생성 → 투두 확인 (F-01, F-02, F-03, F-04)

```
채널 진입
    │
    ▼
[📁 업로드] 클릭
    │
    ▼
파일 선택 다이얼로그
    │
    ├── 지원 형식 아님 → 오류 메시지 → (종료)
    │
    ▼
업로드 프로그레스 바 표시
    │
    ▼
STT 변환 중 상태 표시 ("AI가 회의록을 생성 중입니다...")
    │
    ▼
채널에 회의록 카드 자동 게시 (F-07)
    │
    ├── [회의록 보기 →] 클릭 → 회의록 상세 페이지
    │       │
    │       ▼
    │   참여자별 투두 확인 (F-03)
    │   투두 완료 처리 / 마감일 수정 (F-05)
    │
    └── 홈 대시보드 → "내 할 일"에 자동 반영 (F-04)
```

---

### Flow B — 인앱 녹음 → 회의록 생성 (F-08, F-02, F-03)

```
채널 진입
    │
    ▼
[🎙️ 녹음] 클릭
    │
    ├── 마이크 권한 미허용 → 권한 요청 안내 모달 → (종료)
    │
    ▼
녹음 패널 열림 (경과 시간 카운터 시작)
    │
    ▼
[■ 정지] 클릭
    │
    ▼
자동으로 Flow A의 "STT 변환" 단계로 이어짐
```

---

### Flow C — 채팅 메시지 전송 (F-06)

```
채널 진입
    │
    ▼
메시지 입력창 클릭 → 텍스트/이모지 입력
    │
    ▼
[전송] 클릭 또는 Enter
    │
    ▼
채널 내 모든 참여자 화면에 실시간 표시 (WebSocket)
```

---

### Flow D — 홈에서 투두 관리 (F-04, F-05)

```
홈 대시보드 진입
    │
    ▼
"내 할 일" 섹션에서 투두 확인
    │
    ├── 체크박스 클릭 → 완료 처리 → "완료된 항목"으로 이동
    │
    └── 투두 행 클릭 → 마감일 날짜 피커 → 저장
```

## 5. 데이터 오브젝트 구조

P0 화면 설계를 뒷받침하는 핵심 데이터 오브젝트입니다.

### User
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 |
| name | string | 표시 이름 |
| email | string | 이메일 |
| avatar_url | string | 프로필 이미지 URL |

### Channel
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 |
| name | string | 채널명 |
| members | User[] | 채널 멤버 목록 |

### Message
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 |
| channel_id | string | 소속 채널 |
| sender | User | 발신자 |
| content | string | 텍스트 내용 |
| type | `text` \| `meeting_card` | 일반 메시지 vs 회의록 카드 |
| meeting_id | string? | type이 meeting_card일 때 연결 회의록 ID |
| created_at | timestamp | 전송 시각 |

### Meeting (회의록)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 |
| channel_id | string | 소속 채널 |
| title | string | 회의명 (AI 자동 생성 또는 입력) |
| duration_sec | number | 녹음 길이 (초) |
| summary | string | AI 요약문 |
| discussion_points | string[] | 주요 논의 사항 |
| decisions | string[] | 결정 사항 |
| created_at | timestamp | 생성 일시 |

### Todo
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 |
| meeting_id | string | 출처 회의록 ID |
