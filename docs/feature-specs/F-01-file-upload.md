# 기능 정의서: F-01 — 녹음 파일 업로드

**기능 ID**: F-01
**우선순위**: P0
**연관 기능**: F-02 (AI 회의록 생성), F-08 (인앱 녹음)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

팀원들은 외부 도구(Zoom, 녹음 앱 등)로 녹음한 회의 파일을 MeetSync에 업로드해 AI 분석을 시작하고 싶지만, 해당 파일을 시스템으로 가져오는 표준화된 진입점이 없다. 이 기능이 없으면 AI 회의록 생성 파이프라인(F-02) 전체가 작동하지 않으므로 제품의 핵심 가치 제공이 불가능하다.

---

## 2. 목표

- 업로드 시도 대비 성공률 **80% 이상** 달성 (출시 후 1주 기준)
- 지원 포맷 외 파일 업로드 시도 중 사용자 이탈률 **20% 미만** 유지 (명확한 오류 안내로 재시도 유도)
- 업로드 완료 후 F-02 파이프라인 자동 시작까지 **3초 이내** 전환

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 영상 파일의 화면 영상 재생 | 녹음 음성 추출이 목적이므로 영상 뷰어 불필요 |
| URL 링크로 파일 가져오기 | 외부 스토리지 인증 복잡도 높음, v2 이후 |
| 복수 파일 동시 업로드 | MVP 단계에서 단일 파일 업로드만 지원 |
| 파일 업로드 후 원본 다운로드 제공 | 보안 정책 미결(Q-3) 상태, v2 이후 |

---

## 4. 사용자 스토리

- **팀 리더로서**, 외부 도구로 녹음한 회의 파일을 MeetSync 채널에 업로드하고 싶다. 그래야 AI가 자동으로 회의록을 만들어줄 수 있기 때문이다.
- **팀원으로서**, 내가 녹음한 파일이 올바르게 업로드되고 있는지 진행 상황을 실시간으로 확인하고 싶다. 그래야 업로드가 완료되기 전에 화면을 이탈하는 실수를 방지할 수 있기 때문이다.
- **팀 리더로서**, 지원하지 않는 형식의 파일을 업로드하려 할 때 명확한 안내를 받고 싶다. 그래야 어떤 형식으로 변환해야 하는지 즉시 알 수 있기 때문이다.

---

## 5. 기능 요구사항 및 인수 기준

### 5.1 파일 선택

**Given/When/Then**

- Given 사용자가 채널 상세 화면에 있을 때
- When [📁 업로드] 버튼을 클릭하면
- Then OS 기본 파일 선택 다이얼로그가 열리고, 선택 가능한 파일 형식이 `.mp3, .mp4, .wav, .m4a`로 필터링되어 표시된다

**인수 기준 체크리스트**

- [ ] 지원 형식: `.mp3`, `.mp4`, `.wav`, `.m4a`
- [ ] 파일 크기 제한: 최대 500MB
- [ ] 500MB 초과 파일 선택 시 → 업로드 시작 전 "파일 크기가 너무 큽니다 (최대 500MB)" 오류 표시
- [ ] 지원하지 않는 확장자 선택 시 → "지원하지 않는 형식입니다. mp3, mp4, wav, m4a 파일을 사용해 주세요" 안내
- [ ] 드래그앤드롭으로도 파일 업로드 가능

### 5.2 업로드 진행 표시

- [ ] 업로드 시작 즉시 채널 입력창 영역에 프로그레스 바 표시 (0~100%)
- [ ] 업로드 중 파일명과 파일 크기 함께 표시 (예: "주간미팅_0324.mp3 · 24.3MB")
- [ ] 업로드 중 [취소] 버튼 제공 → 클릭 시 업로드 중단 및 임시 파일 삭제
- [ ] 업로드 완료 시 자동으로 F-02 AI 처리 파이프라인 트리거

### 5.3 오류 처리

| 오류 상황 | 사용자에게 표시할 메시지 |
|---|---|
| 네트워크 연결 끊김 | "네트워크 오류가 발생했습니다. 연결을 확인하고 다시 시도해 주세요." |
| 서버 오류 (5xx) | "업로드 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." |
| 파일 손상 | "파일을 읽을 수 없습니다. 파일이 손상되었을 수 있습니다." |

- [ ] 오류 발생 시 [다시 시도] 버튼 제공
- [ ] 오류 후 파일 재선택 없이 동일 파일로 재시도 가능

---

## 6. 기술 고려사항

- **멀티파트 업로드**: 대용량 파일(>100MB)은 청크 단위로 분할 업로드하여 네트워크 오류 시 재개 가능하도록 구현 권장
- **임시 저장 위치**: 업로드 완료 파일은 서버 임시 스토리지에 저장 후 STT 처리 완료 시 보존 여부를 정책에 따라 결정 (오픈 이슈 Q-3)
- **MIME 타입 검증**: 확장자 외 실제 MIME 타입도 서버 사이드에서 검증하여 우회 방지
