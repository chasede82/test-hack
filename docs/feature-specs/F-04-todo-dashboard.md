# 기능 정의서: F-04 — 개인 투두 대시보드

**기능 ID**: F-04
**우선순위**: P0
**연관 기능**: F-03 (투두 추출), F-05 (투두 관리)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

여러 채널에서 다수의 회의가 진행되면 각 회의록에 흩어진 자신의 할 일을 파악하기 위해 회의록을 하나씩 열어봐야 한다. 이로 인해 중요한 투두 항목을 놓치거나 마감을 초과하는 사례가 발생한다. 개인 대시보드는 나에게 할당된 모든 투두를 한눈에 볼 수 있는 중앙화된 공간을 제공해 후속 업무 누락을 방지한다.

---

## 2. 목표

- 로그인 후 첫 화면에서 **내 투두 전체를 1개의 화면**으로 확인 가능
- 투두 대시보드 **일간 방문율 60% 이상** (WAU 대비, 출시 후 4주)
- "어느 회의에서 나온 할 일인지" 출처를 **100%** 표시 (맥락 손실 방지)

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 팀 전체 투두 통합 보기 | 프라이버시 이슈, 관리자 전용 v2 기능으로 분리 |
| 투두 수동 생성 (회의 외 출처) | MVP는 회의록 기반 투두만 지원 |
| 캘린더 뷰 (날짜 기반 시각화) | P2 이후, 마감일 정보가 충분히 쌓인 이후 도입 |
| 투두 담당자 재지정 | P1(F-10)으로 분리 |

---

## 4. 사용자 스토리

- **팀원으로서**, 로그인 후 메인 화면에서 내 모든 할 일을 한 번에 확인하고 싶다. 그래야 여러 회의록을 열지 않고 오늘 해야 할 업무를 바로 파악할 수 있기 때문이다.
- **팀원으로서**, 각 투두가 어느 회의에서 나왔는지 출처를 바로 확인하고 싶다. 그래야 맥락을 모를 때 해당 회의록으로 바로 이동해 내용을 확인할 수 있기 때문이다.
- **팀원으로서**, 마감일이 가까운 투두가 상단에 표시되길 원한다. 그래야 긴급한 업무를 놓치지 않을 수 있기 때문이다.
- **팀원으로서**, 완료된 투두와 진행 중인 투두를 분리해서 보고 싶다. 그래야 이미 처리한 일을 다시 볼 필요 없이 남은 일에만 집중할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 대시보드 진입

- [ ] 로그인 완료 후 홈 대시보드가 기본 랜딩 페이지
- [ ] 사이드바 "🏠 홈" 아이콘 클릭으로 언제든 돌아올 수 있음

### 5.2 "내 할 일" 섹션 — 진행 중 목록

**표시 정보 (투두 1건당)**

| 필드 | 표시 형식 |
|---|---|
| 완료 체크박스 | ☐ (미완료) / ☑ (완료) |
| 투두 내용 | 텍스트 한 줄 |
| 마감일 | "3/26" 또는 "마감일 미정" |
| 마감 임박 표시 | 오늘 포함 3일 이내 → 마감일 빨간색 표시 |
| 출처 회의명 | 예: "주간 팀 미팅 · 3/24" |

**Given/When/Then**

- Given 로그인한 사용자가 홈 대시보드에 진입하면
- When 페이지가 로드될 때
- Then 나에게 할당된 미완료 투두 전체가 마감일 오름차순으로 표시된다 (마감일 미정은 최하단)

**인수 기준**

- [ ] 미완료 투두 전체 표시 (개수 제한 없음)
- [ ] 마감일 오름차순 정렬, 마감일 동일 시 생성일 오름차순 정렬
- [ ] 마감일 미정 항목은 정렬 최하단
- [ ] 투두가 없을 경우 "아직 할 일이 없습니다 🎉" 빈 상태 메시지 표시

### 5.3 "내 할 일" 섹션 — 완료 목록

- [ ] "완료된 항목 보기 ▾" 버튼으로 접기/펼치기 (기본: 접힌 상태)
- [ ] 완료 투두는 완료 처리 일시 내림차순 표시
- [ ] 완료 투두에는 취소선 + 회색 텍스트 스타일 적용

### 5.4 출처 회의록 연결

- [ ] 투두의 출처 회의명 클릭 시 해당 회의록 상세 페이지로 이동
- [ ] 출처 회의록이 삭제된 경우 → "삭제된 회의" 텍스트로 표시 (링크 비활성)

### 5.5 실시간 반영

**Given/When/Then**

- Given 사용자가 회의록 상세 페이지 또는 다른 세션에서 투두를 완료 처리하면
- When 홈 대시보드로 이동하면
- Then 완료 처리된 투두가 "진행 중" 목록에서 제거되고 "완료" 목록에 표시된다

- [ ] 페이지 새로고침 없이도 최신 상태 반영 (또는 진입 시 최신 데이터 fetch)

### 5.6 최근 회의록 섹션

- [ ] 내가 속한 채널의 최근 회의록 최대 5건 표시
- [ ] 각 항목: 회의명, 날짜, 소속 채널명
- [ ] 클릭 시 회의록 상세 페이지 이동
- [ ] "전체 보기 →" 클릭 시 회의록 목록 페이지 이동
