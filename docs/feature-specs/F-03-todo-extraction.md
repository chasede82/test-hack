# 기능 정의서: F-03 — 참여자별 투두 자동 추출

**기능 ID**: F-03
**우선순위**: P0
**연관 기능**: F-02 (AI 회의록 생성), F-04 (투두 대시보드), F-05 (투두 관리)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

회의에서 논의된 액션 아이템이 참여자별로 명확하게 정리되지 않아 후속 업무가 누락되거나 담당자가 서로 다른 기억을 갖는 문제가 반복된다. AI 회의록(F-02) 생성 과정에서 참여자별 할 일을 자동으로 분류·추출하면, 회의 직후 별도의 정리 작업 없이 각자 담당 업무를 즉시 확인할 수 있다.

---

## 2. 목표

- 회의록당 평균 **3건 이상**의 투두 자동 추출 (내용이 있는 회의 기준)
- 추출된 투두 중 담당자 매핑 성공률 **70% 이상** (사용자 수동 수정 최소화)
- 투두 생성 후 **72시간 이내** 담당자의 확인율 **70% 이상**

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 투두 담당자 재지정 | P1(F-10)으로 분리 |
| 투두 우선순위 자동 분류 | AI 판단 오류 위험, v2 이후 |
| 외부 프로젝트 툴(Jira 등)로 내보내기 | P2(F-15)로 분리 |
| 투두 마감일 AI 자동 확정 | AI 추론 신뢰도 낮음, 추정값으로만 제공 |

---

## 4. 사용자 스토리

- **팀 리더로서**, 회의가 끝나면 AI가 "누가 무엇을 해야 하는지"를 자동으로 정리해 주길 원한다. 그래야 별도 회의록 작성 없이 팀원들에게 각자의 할 일을 즉시 공유할 수 있기 때문이다.
- **팀원으로서**, 회의에서 내 이름이 언급된 할 일이 내 투두 대시보드에 자동으로 나타나길 원한다. 그래야 회의 후 따로 메모를 확인하지 않아도 내 업무를 파악할 수 있기 때문이다.
- **팀원으로서**, AI가 추출한 투두 내용이 실제 회의 맥락과 다를 경우 이를 수정할 수 있길 원한다. 그래야 잘못된 업무를 그대로 진행하는 상황을 방지할 수 있기 때문이다.

---

## 5. 기능 요구사항 및 인수 기준

### 5.1 투두 추출 로직

**Given/When/Then**

- Given F-02 AI 회의록 생성이 완료되면
- When 시스템이 "참여자별 할 일" 섹션을 파싱하면
- Then 발화자 이름과 액션 아이템이 매핑된 투두 레코드가 DB에 생성된다

**인수 기준**

- [ ] 투두 1건당 포함 정보: 내용, 담당자명, 마감일(추정 또는 `미정`), 출처 회의록 ID
- [ ] 마감일은 회의 내용에서 언급된 경우 추출, 미언급 시 `미정`으로 저장
- [ ] 동일 인물에게 복수의 할 일이 있을 경우 각각 별도 투두로 분리
- [ ] 담당자가 명확하지 않은 액션 아이템은 `담당자 미지정` 상태로 저장

### 5.2 담당자 계정 매핑

**문제**: AI가 추출하는 담당자는 "발화된 이름(텍스트)"이며, 이를 실제 MeetSync 사용자 계정과 연결해야 한다.

**매핑 전략 (MVP 기준)**

| 단계 | 방식 |
|---|---|
| 1차 | 채널 멤버 목록의 표시 이름(display_name)과 정확히 일치하면 자동 매핑 |
| 2차 | 포함 관계 매칭 (예: "준혁" → "김준혁" 계정으로 매핑) |
| 3차 | 매핑 실패 시 `담당자 미지정` 상태로 저장, 회의록 상세에서 수동 지정 가능 |

- [ ] 매핑 성공 시 해당 사용자의 투두 대시보드(F-04)에 즉시 반영
- [ ] 매핑 실패 투두는 회의록 상세 페이지에 "⚠️ 담당자를 확인해 주세요" 배지로 표시

### 5.3 투두 내용 편집 (기본)

- [ ] 회의록 상세 화면의 투두 항목에서 내용(텍스트) 인라인 편집 가능
- [ ] 마감일 수정 가능 (날짜 피커)
- [ ] 편집 저장 시 해당 사용자의 대시보드(F-04)에도 즉시 반영

### 5.4 빈 결과 처리

- [ ] 회의 내용에서 액션 아이템이 감지되지 않은 경우: "이 회의에서 추출된 할 일이 없습니다" 표시
- [ ] 투두 0건이어도 회의록 자체는 정상 생성됨 (F-02 영향 없음)

## 6. 기술 고려사항

- **추출 방식**: F-02의 GPT 프롬프트에서 회의록 생성과 투두 추출을 단일 호출로 처리 (별도 API 호출 ✕, 비용·지연 최소화)
- **GPT 응답 포맷**: JSON 구조화 응답 사용

```json
{
  "todos": [
    {
      "assignee_name": "챈들러",
      "content": "기획안 초안 작성",
      "due_date": "2026-03-26"
    },
    {
      "assignee_name": "수연",
      "content": "목업 디자인 완성",
      "due_date": null
    }
  ]
}
```

- **매핑 실패 처리**: 미매핑 투두는 `assignee_id = null`, `assignee_raw_name = "수연"` 형태로 저장
