# 기능 정의서: F-06 — 실시간 채팅

**기능 ID**: F-06
**우선순위**: P0
**연관 기능**: F-07 (회의록 카드 공유)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

팀은 회의 전·중·후의 맥락 있는 대화와 회의 결과물(회의록, 투두)을 별도의 채널에서 관리하고 있어, 논의 흐름이 이메일·메신저·문서 등 여러 곳에 분산된다. MeetSync의 실시간 채팅은 회의 결과물과 팀 소통이 동일 공간에서 이루어지도록 하여, 회의 전후 맥락 손실을 방지하는 협업의 기반 레이어다.

---

## 2. 목표

- 메시지 전송 후 채널 내 모든 참여자 화면 반영 지연 **1초 이내** (정상 네트워크 기준)
- 채널당 일간 메시지 전송 수 **평균 20건 이상** (출시 후 4주, WAU 팀 기준)
- 채팅 관련 오류(메시지 미전송, 중복 전송 등) 발생률 **1% 미만**

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 파일 첨부 | P1(F-11)으로 분리 |

| 읽음 확인(Read receipt) | MVP 이후 |

## 4. 사용자 스토리

- **팀원으로서**, 채널에서 실시간으로 텍스트 메시지를 주고받고 싶다. 그래야 회의 전후 논의를 이메일 없이 한 곳에서 처리할 수 있기 때문이다.
- **팀원으로서**, 내가 보낸 메시지가 상대방 화면에 즉시 나타나는 것을 확인하고 싶다. 그래야 소통이 실시간으로 이루어지고 있다는 확신을 가질 수 있기 때문이다.
- **팀원으로서**, 이모지를 메시지에 포함해 감정을 표현하고 싶다. 그래야 텍스트 메시지만으로 딱딱한 소통을 보완할 수 있기 때문이다.
- **팀원으로서**, 채널에 입장했을 때 이전 대화 내역을 처음부터 볼 수 있길 원한다. 그래야 내가 없는 동안 어떤 논의가 있었는지 파악할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 메시지 전송

**Given/When/Then**

- Given 사용자가 채널 하단 입력창에 텍스트를 입력하고
- When [전송] 버튼을 클릭하거나 Enter 키를 누르면
- Then 메시지가 채널 피드에 즉시 표시되고, WebSocket을 통해 동일 채널의 모든 참여자에게 실시간으로 전달된다

**인수 기준**

- [ ] 텍스트 입력 후 Enter 또는 전송 버튼으로 전송
- [ ] Shift+Enter는 줄바꿈 (전송 ✕)
- [ ] 빈 메시지(공백만 있는 경우) 전송 불가 — 전송 버튼 비활성화
- [ ] 메시지 최대 길이: 2,000자 (초과 입력 시 입력 차단, 잔여 글자 수 표시)
- [ ] 전송 완료 후 입력창 초기화
- [ ] 이모지 입력 지원 (OS 이모지 피커 또는 이모지 버튼)

### 5.2 실시간 수신

- [ ] 채널에 접속 중인 모든 참여자는 새 메시지를 **1초 이내** 수신
- [ ] 새 메시지 수신 시 채팅 피드가 최신 메시지로 자동 스크롤 (단, 사용자가 위로 스크롤 중이면 자동 스크롤 ✕)
- [ ] 사용자가 위로 스크롤 중일 때 새 메시지가 오면 "새 메시지 보기 ↓" 버튼 표시 → 클릭 시 최하단으로 이동

### 5.3 메시지 피드 표시

**메시지 1건당 표시 정보**

| 필드 | 표시 형식 |
|---|---|
| 발신자 프로필 이미지 | 작은 원형 아바타 |
| 발신자 이름 | 굵은 텍스트 |
| 전송 시각 | "오전 10:32" 형식 |
| 메시지 내용 | 텍스트 (이모지 포함) |

- [ ] 동일 발신자가 연속으로 메시지를 보낼 경우 (5분 이내): 아바타·이름 생략, 시각만 표시 (그룹핑)
- [ ] 날짜가 바뀌는 시점에 날짜 구분선 표시 (예: "─── 2026년 3월 24일 ───")

### 5.4 채팅 이력 로드

**Given/When/Then**

- Given 사용자가 채널에 처음 진입하면
- When 채널 화면이 로드되면
- Then 최근 50건의 메시지를 최신순으로 표시하고 피드 최하단에 위치한다

- [ ] 피드 최상단으로 스크롤 시 이전 메시지 50건 추가 로드 (무한 스크롤 위 방향)
- [ ] 이전 메시지 로드 중 로딩 스피너 표시
- [ ] 더 이상 불러올 메시지가 없으면 "채널의 시작입니다 🎉" 표시

### 5.5 연결 상태 처리

| 상태 | UI 표시 |
|---|---|
| 연결 중 | 상단 배너: "연결 중..." |
| 연결 끊김 | 상단 배너: "연결이 끊어졌습니다. 재연결 중..." (자동 재연결 시도) |
| 재연결 성공 | 배너 사라짐, 끊어진 동안의 메시지 자동 수신 |

- [ ] 오프라인 상태에서 전송 시도 시 "인터넷 연결을 확인해 주세요" 경고 표시
- [ ] 자동 재연결: 1초 → 3초 → 5초 간격 지수 백오프

## 6. 기술 고려사항

- **WebSocket 라이브러리**: Socket.io (v4) 또는 Supabase Realtime 권장
  - Socket.io: 폴링 폴백 지원, 연결 안정성 높음
  - Supabase Realtime: DB 변경 이벤트 기반, 별도 서버 불필요
- **메시지 ID**: 클라이언트에서 임시 ID 생성 → 서버 확인 후 영구 ID로 교체 (낙관적 전송)
- **동시 접속 규모**: MVP 기준 채널당 최대 50명 동시 접속 고려
- **메시지 저장**: 모든 메시지는 DB에 영구 저장 (채팅 이력 조회용)
