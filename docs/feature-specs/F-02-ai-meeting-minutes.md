# 기능 정의서: F-02 — AI 회의록 자동 생성

**기능 ID**: F-02
**우선순위**: P0
**연관 기능**: F-01 (파일 업로드), F-03 (투두 추출), F-07 (채널 카드 공유), F-08 (인앱 녹음)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

회의 후 공식 회의록을 작성하는 데 팀 평균 30~60분이 소요되며, 이 시간 동안 회의 내용이 기억에서 희미해지고 핵심 결정 사항이 누락되는 경우가 빈번하다. 이 기능은 업로드된 녹음 파일을 STT로 변환하고 GPT가 요약·구조화하여 5분 이내에 회의록 초안을 자동 생성함으로써, 회의록 작성의 인적 부담을 제거하는 것을 목표로 한다.

---

## 2. 목표

- AI 회의록 생성 파이프라인 **성공률 95% 이상** (STT + GPT 처리 기준, 출시 후 4주)
- 녹음 파일 업로드 완료 후 회의록 초안 채널 게시까지 **5분 이내** (60분 분량 파일 기준)
- 생성된 회의록의 핵심 결정 사항 누락률 **10% 미만** (초기 사용자 인터뷰 기준)

---

## 4. 사용자 스토리

- **팀 리더로서**, 녹음 파일 업로드 후 AI가 자동으로 회의록을 생성해 주길 원한다. 그래야 회의가 끝난 직후 수동으로 정리할 필요 없이 바로 팀과 공유할 수 있기 때문이다.
- **팀원으로서**, 생성된 회의록이 "요약 / 주요 논의 사항 / 결정 사항" 형태로 구조화되어 있길 원한다. 그래야 긴 회의 내용을 빠르게 파악할 수 있기 때문이다.
- **팀 리더로서**, AI 처리 진행 상태를 채널에서 실시간으로 확인하고 싶다. 그래야 언제 회의록을 볼 수 있는지 예측할 수 있기 때문이다.
- **팀원으로서**, AI 처리가 실패했을 때 오류 원인과 재시도 방법을 안내받고 싶다. 그래야 혼란 없이 다음 행동을 취할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 처리 파이프라인 자동 시작

- [ ] F-01 업로드 완료 또는 F-08 녹음 정지 즉시 파이프라인 자동 시작 (사용자 별도 액션 불필요)
- [ ] 파이프라인 시작과 동시에 채널에 처리 상태 메시지 게시

### 5.2 처리 상태 표시

| 단계 | 채널 표시 메시지 |
|---|---|
| 업로드 완료 | "📤 녹음 파일 업로드 완료. AI가 분석을 시작합니다..." |
| STT 변환 중 | "🎙️ 음성을 텍스트로 변환하는 중입니다..." |
| 회의록 생성 중 | "🤖 AI가 회의록을 작성하는 중입니다..." |
| 완료 | 회의록 카드 자동 게시 (F-07) |
| 오류 | "⚠️ 회의록 생성에 실패했습니다. [다시 시도]" |

- [ ] 처리 중 상태 메시지는 완료/실패 시 자동으로 회의록 카드로 교체됨 (별도 메시지 추가 ✕)

### 5.3 회의록 구조 (AI 생성 결과)

```
제목: [AI 자동 추론 또는 날짜+채널명 조합]
날짜: YYYY-MM-DD
소요 시간: N분

## 📝 요약
2~4문장 핵심 요약

## 💬 주요 논의 사항
- 논의 항목 1
- 논의 항목 2
...

## ✅ 결정 사항
- 결정된 내용 1
- 결정된 내용 2
...

## 👥 참여자별 할 일
→ F-03에서 생성
```

**인수 기준**

- [ ] 요약은 2~4문장, 전체 회의 맥락을 포함
- [ ] 주요 논의 사항은 최소 1개 이상 (녹음 내용이 있는 경우)
- [ ] 결정 사항이 없는 경우 해당 섹션에 "결정된 사항 없음" 표시
- [ ] 회의록 제목: AI가 대화 내용에서 자동 추론, 추론 불가 시 `{날짜} {채널명} 회의`로 폴백

### 5.4 오류 처리

**Given/When/Then**

- Given STT API 또는 GPT API 호출이 실패하면
- When 시스템이 오류를 감지하면
- Then 채널에 "⚠️ 회의록 생성에 실패했습니다. [다시 시도]" 메시지가 표시되고, [다시 시도] 클릭 시 동일 파일로 파이프라인 재실행된다

- [ ] 최초 실패 시 자동으로 1회 재시도 (사용자 미인지)
- [ ] 자동 재시도도 실패 시 채널에 오류 메시지 + [다시 시도] 버튼 노출
- [ ] 재시도는 동일 업로드 파일 기준으로 실행 (파일 재업로드 불필요)

## 6. 기술 고려사항

- **STT**: OpenAI Whisper API 권장 (한국어 지원, 비용 효율)
  - 모델: `whisper-1`
  - 최대 처리 파일: 25MB 제한 → 초과 시 파일 분할 처리 필요
  - 25MB 초과 파일은 청크 분할 → 순차 STT → 텍스트 병합 후 GPT 처리
- **회의록 생성**: GPT-4o
  - 시스템 프롬프트에 구조(요약/논의/결정) 포맷 명시
  - 최대 context window 고려: 1시간 분량 회의 = 약 8,000~12,000 토큰
- **비동기 처리**: 파이프라인은 백그라운드 큐(Worker)에서 실행, 완료 시 WebSocket으로 채널에 푸시
- **처리 시간 예상**: 60분 분량 기준 Whisper ~2분 + GPT ~30초 = 약 2~3분

## 7. GPT 프롬프트 설계 가이드

```
[시스템 프롬프트]
당신은 한국어 회의록 작성 전문가입니다.
아래 회의 음성 전사 내용을 바탕으로 다음 형식에 맞게 회의록을 작성하세요:

1. 제목: 회의 주제를 한 줄로 요약
2. 요약: 전체 회의를 2~4문장으로 핵심 요약
3. 주요 논의 사항: 논의된 주제별 불릿 포인트
4. 결정 사항: 회의에서 확정된 내용만 불릿 포인트로 정리
5. 참여자별 할 일: 발화자 이름 기준으로 액션 아이템 분류

- 없는 내용은 "해당 없음"으로 표기
- 추측하지 말고 전사 내용에 근거해서만 작성
```
