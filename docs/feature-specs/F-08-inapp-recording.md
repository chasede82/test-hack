# 기능 정의서: F-08 — 인앱 실시간 녹음

**기능 ID**: F-08
**우선순위**: P0
**연관 기능**: F-01 (파일 업로드), F-02 (AI 회의록 생성)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

회의 참여자가 외부 녹음 앱을 별도로 실행하고, 녹음 완료 후 파일을 저장하고, 다시 MeetSync에 업로드하는 3단계 절차는 진입 장벽이 높다. 특히 즉흥적으로 시작된 회의나 짧은 동기화 미팅에서는 외부 도구를 준비할 여유가 없다. 인앱 실시간 녹음은 MeetSync 안에서 바로 녹음을 시작할 수 있게 하여 이 마찰을 제거하고, AI 분석 파이프라인으로의 자동 전환을 가능하게 한다.

---

## 2. 목표

- 인앱 녹음 기능 사용률: 녹음 방식 중 **인앱 녹음 비중 40% 이상** (출시 4주 기준, 파일 업로드 대비)
- 녹음 시작에서 AI 파이프라인 시작까지 **탭 이동 없이** 완결
- 마이크 권한 요청 후 거절한 사용자의 **재시도율 30% 이상** (명확한 안내로 재동의 유도)

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 다중 마이크 채널 녹음 (각 참여자 별도 트랙) | 하드웨어 접근 제한, v2 이후 |
| 녹음 중 실시간 파형(Waveform) 시각화 | 구현 복잡도, MVP에서는 경과 시간 표시로 대체 |
| 녹음 파일 다운로드 제공 | 보안 정책 미결(Q-3), v2 이후 |
| 배경 잡음 제거(노이즈 캔슬링) | 브라우저 API 수준에서 처리, 고품질 처리는 v2 |
| 화상 회의 중 화면 녹화 | 영상 캡처는 별도 인프라 필요 |

## 4. 사용자 스토리

- **팀 리더로서**, 회의가 시작될 때 MeetSync를 열고 바로 녹음 버튼을 눌러 녹음을 시작하고 싶다. 그래야 별도 앱을 준비할 필요 없이 즉시 회의 기록을 시작할 수 있기 때문이다.
- **팀원으로서**, 녹음이 진행 중임을 화면에서 직관적으로 확인하고 싶다. 그래야 녹음이 제대로 되고 있는지 불안해하지 않아도 되기 때문이다.
- **팀 리더로서**, 회의 종료 후 정지 버튼을 누르면 AI가 자동으로 회의록을 만들어 주길 원한다. 그래야 녹음 파일을 따로 저장하고 업로드하는 번거로움 없이 바로 결과를 받을 수 있기 때문이다.
- **팀 리더로서**, 회의 중 잠깐 녹음을 멈출 수 있길 원한다. 그래야 회의와 무관한 대화가 회의록에 포함되지 않도록 조절할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 녹음 패널 진입

**Given/When/Then**

- Given 사용자가 채널 상세 화면에 있을 때
- When [🎙️ 녹음] 버튼을 클릭하면
- Then 녹음 패널이 슬라이드업 형태로 화면 하단에 표시된다

- [ ] 녹음 패널은 채팅 화면 위에 오버레이로 표시 (별도 페이지 이동 ✕)
- [ ] 패널 내부에 파일 업로드 탭 전환 가능 (F-01과 동일 진입점 공유)
- [ ] [✕ 닫기] 버튼으로 패널 닫기 가능 (녹음 중일 때는 확인 모달 표시)

### 5.2 마이크 권한 처리

**Given/When/Then**

- Given 사용자가 처음으로 [🎙️ 녹음] 버튼을 클릭하면
- When 브라우저 마이크 권한이 허용되지 않은 상태이면
- Then 브라우저 권한 요청 팝업이 표시되고, 거절 시 안내 모달이 표시된다

**권한 거절 안내 모달 내용**

```
🎙️ 마이크 접근이 필요합니다

녹음 기능을 사용하려면 브라우저에서 마이크 권한을 허용해야 합니다.

[브라우저 설정 방법 보기]    [닫기]
```

- [ ] 권한 허용 상태에서 클릭: 즉시 녹음 시작 (추가 팝업 ✕)
- [ ] 권한 거절 상태에서 클릭: 안내 모달 표시
- [ ] 안내 모달 내 "브라우저 설정 방법 보기" → 브라우저별 권한 설정 안내 표시 (Chrome/Safari/Edge 분기)

### 5.3 녹음 제어

**상태 전이**

```
[대기] ──────────────── [🔴 녹음 중] ──────── [일시정지] ────── [재개 → 녹음 중]
  ↑ 시작                  ↓ 정지                  ↓ 정지
                        [저장 완료 → F-02 자동 시작]
```

**인수 기준**

- [ ] 녹음 시작: [● 녹음 시작] 버튼 → 빨간색 점 깜빡임 + 경과 시간 카운터 (00:00:00) 시작
- [ ] 일시정지: [⏸ 일시정지] 버튼 → 경과 시간 정지, "일시정지됨" 텍스트 표시
- [ ] 재개: [▶ 재개] 버튼 → 경과 시간 이어서 카운트
- [ ] 정지: [■ 정지] 버튼 → 확인 모달 표시 후 저장 진행
- [ ] 정지 확인 모달: "녹음을 종료하고 AI 회의록 생성을 시작할까요?" / [종료 및 생성] / [계속 녹음]
- [ ] 최소 녹음 시간 제한 없음 (1초짜리 녹음도 저장 가능)
- [ ] 최대 녹음 시간: 3시간 (초과 시 자동 정지 + 안내 메시지)

### 5.4 저장 및 AI 파이프라인 자동 시작

**Given/When/Then**

- Given 사용자가 [종료 및 생성] 버튼을 클릭하면
- When 시스템이 녹음 파일을 서버에 전송하면
- Then F-01의 업로드 프로그레스 표시 없이 바로 F-02 처리 상태("🎙️ 음성을 텍스트로 변환하는 중...")로 전환된다

- [ ] 인앱 녹음 완료 파일은 서버에 전송 후 F-02 파이프라인 자동 실행
- [ ] 파일 전송 중 실패 시 "녹음 파일 저장 중 오류가 발생했습니다. 다시 시도해 주세요." 표시
- [ ] 녹음 파일 형식: WebM (브라우저 MediaRecorder 기본값) → 서버에서 mp3/wav로 변환 후 Whisper 처리

### 5.5 마이크 장치 선택

- [ ] 기본 마이크(OS 기본 입력 장치) 자동 선택
- [ ] 복수 마이크 장치 연결 시 [변경] 버튼으로 목록에서 선택 가능
- [ ] 마이크 장치 선택은 녹음 시작 전에만 가능 (녹음 중 변경 ✕)

## 6. 기술 고려사항

- **API**: 브라우저 MediaRecorder API 사용 (Web Audio API는 MVP 제외)
  - `navigator.mediaDevices.getUserMedia({ audio: true })`
  - 지원 브라우저: Chrome 47+, Firefox 25+, Safari 14.1+ (Edge 79+)
- **파일 포맷**: MediaRecorder 기본 출력 포맷은 브라우저마다 다름
  - Chrome: `audio/webm;codecs=opus`
  - Safari: `audio/mp4`
  - 서버에서 ffmpeg를 통해 통일된 포맷(wav)으로 변환 후 Whisper 처리
- **메모리 관리**: 장시간 녹음 시 브라우저 메모리 부족 방지를 위해 일정 간격(예: 30초)마다 청크를 서버로 전송하는 스트리밍 업로드 구현 권장
- **Whisper 파일 크기 제한**: 25MB 초과 시 분할 처리 (F-02 기술 고려사항 참조)
