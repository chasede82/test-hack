# 기능 정의서: F-07 — 채널 내 회의록 카드 자동 공유

**기능 ID**: F-07
**우선순위**: P0
**연관 기능**: F-02 (AI 회의록 생성), F-06 (실시간 채팅)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

AI 회의록이 생성되어도 팀원들이 이를 인지하지 못하면 활용이 이루어지지 않는다. 회의록이 별도 페이지에만 저장되고 채팅 흐름과 단절되면, 팀원들은 직접 회의록 목록을 찾아야 한다는 불편함으로 인해 회의록 확인을 미루거나 놓치게 된다. 채팅 채널에 회의록 카드를 자동으로 게시하여 "회의가 끝났다 → 회의록이 생겼다"는 신호를 팀 전체가 동시에 인지할 수 있도록 한다.

---

## 2. 목표

- 회의록 생성 완료 후 채널 카드 게시까지 **5초 이내**
- 회의록 카드 게시 후 **1시간 이내** 채널 멤버의 카드 클릭률 **50% 이상** (출시 후 4주)
- 카드를 통한 회의록 상세 페이지 유입이 전체 회의록 조회의 **70% 이상** 차지

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 회의록 카드에서 직접 투두 완료 처리 | 카드는 진입점 역할, 관리는 대시보드/상세 페이지에서 수행 |
| 카드 핀(고정) 기능 | P1 이후 |
| 카드를 외부 링크로 공유 | 보안 정책 미결, v2 |
| 회의록 카드 직접 편집 | 카드는 읽기 전용, 편집은 상세 페이지에서 |

---

## 4. 사용자 스토리

- **팀 리더로서**, 회의록 생성이 완료되면 채팅 채널에 자동으로 알림이 올라오길 원한다. 그래야 팀원들에게 별도로 공유 메시지를 보내지 않아도 되기 때문이다.
- **팀원으로서**, 채팅 피드에서 회의록 요약을 카드 형태로 확인하고 싶다. 그래야 회의록 상세 페이지에 들어가지 않아도 핵심 내용을 빠르게 파악할 수 있기 때문이다.
- **팀원으로서**, 카드를 클릭하면 회의록 상세 페이지로 바로 이동하고 싶다. 그래야 투두 확인과 전체 회의 내용을 이어서 볼 수 있기 때문이다.
- **팀원으로서**, 내 이름으로 생성된 투두 수가 카드에 표시되길 원한다. 그래야 회의록을 열기 전에 내가 맡은 일이 있는지 파악할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 자동 게시 트리거

**Given/When/Then**

- Given F-02 AI 회의록 생성 파이프라인이 완료되면
- When 시스템이 완료 이벤트를 감지하면
- Then 해당 채널의 채팅 피드에 회의록 카드 메시지가 자동으로 게시된다

**인수 기준**

- [ ] 사람이 직접 보내는 메시지가 아닌, 시스템 메시지(Bot) 형태로 게시
- [ ] 회의록 생성 완료 이벤트 수신 후 **5초 이내** 카드 게시
- [ ] 카드 게시는 채널에 접속 중인 모든 멤버에게 실시간 수신 (F-06 WebSocket 활용)

### 5.2 카드 디자인 및 표시 정보

```
┌──────────────────────────────────────────────────────┐
│ 🤖 MeetSync AI                                       │
├──────────────────────────────────────────────────────┤
│ 📄 2026-03-24 주간 팀 미팅                            │
│ ─────────────────────────────────────────────────    │
│ 📝 요약                                               │
│ 신규 기능 개발 일정 조율 및 담당자 역할 분담을 논의.  │
│ 다음 스프린트 목표는 4/14 배포로 결정.                │
│ ─────────────────────────────────────────────────    │
│ ✅ 결정 사항 2건   👥 할 일 3건 생성됨               │
│                                      [회의록 보기 →] │
└──────────────────────────────────────────────────────┘
```

**표시 정보 인수 기준**

- [ ] 회의록 제목 (날짜 + 회의명)
- [ ] AI 요약문 (최대 2줄, 초과 시 말줄임표 처리)
- [ ] 결정 사항 건수
- [ ] 생성된 투두 총 건수
- [ ] [회의록 보기 →] 버튼 → 클릭 시 회의록 상세 페이지로 이동
- [ ] 발신자: "🤖 MeetSync AI" (일반 사용자 메시지와 시각적으로 구분)
- [ ] 카드 배경색/테두리로 일반 채팅 메시지와 시각적 구분

### 5.3 처리 중 상태 메시지 → 카드 전환

- [ ] AI 처리 중 표시된 상태 메시지(F-02 5.2 참조)가 완료 시 회의록 카드로 **교체** (새 메시지 추가 ✕)
- [ ] 카드로 전환될 때 부드러운 시각 전환 (페이드 또는 슬라이드)

### 5.4 오류 시 처리

**Given/When/Then**

- Given F-02 파이프라인이 최종 실패하면
- When 시스템이 실패 상태를 감지하면
- Then 처리 중 메시지가 오류 메시지로 교체된다: "⚠️ 회의록 생성에 실패했습니다. [다시 시도]"

- [ ] 오류 카드의 [다시 시도] 버튼 클릭 시 동일 파일로 F-02 파이프라인 재실행
- [ ] 재시도 중에는 다시 처리 중 상태로 전환

## 6. 기술 고려사항

- **메시지 타입**: `type: "meeting_card"` — 일반 메시지와 구분되는 DB 필드 및 렌더러 분리
- **WebSocket 이벤트**: 회의록 생성 완료 시 서버에서 `meeting_card_created` 이벤트를 채널 Room에 broadcast
- **카드 렌더러**: 클라이언트에서 `type === "meeting_card"` 감지 시 카드 컴포넌트 렌더링, 아니면 일반 메시지 컴포넌트 렌더링
- **카드 편집 불가**: 카드는 시스템 생성 메시지이므로 편집·삭제 UI 미제공
