# 기능 정의서: F-05 — 투두 관리 (완료 처리·마감일 수정·삭제)

**기능 ID**: F-05
**우선순위**: P0
**연관 기능**: F-03 (투두 추출), F-04 (투두 대시보드)
**최종 수정일**: 2026-03-24

---

## 1. 문제 정의

AI가 자동 추출한 투두는 초기 상태 그대로 영구히 남아 있으면 실제 업무 진행 상황을 반영하지 못한다. 사용자가 완료 여부를 표시하고 마감일을 조정할 수 있어야 투두 목록이 실제 업무 현황을 반영하는 살아 있는 도구가 된다. 이 기능이 없으면 투두 대시보드(F-04)는 쌓이기만 하는 정적 목록이 되어 사용자가 이탈한다.

---

## 2. 목표

- 생성된 투두 중 **72시간 이내** 완료 처리 또는 마감일 수정 비율 **60% 이상** (출시 후 4주)
- 투두 완료 처리 액션의 **오류율 0%** (데이터 손실 없이 상태 변경)
- 완료된 투두는 "완료 목록"에 보존하여 **감사 추적(audit trail)** 가능

---

## 3. 비목표

| 항목 | 이유 |
|---|---|
| 투두 담당자 재지정 | P1(F-10)으로 분리 |
| 투두 수동 생성 | MVP는 AI 추출 기반, 수동 생성은 v2 |
| 투두 일괄 완료 처리 | 사용 빈도 낮음, 단건 처리만 지원 |
| 투두 완료 취소 (완료 → 미완료 되돌리기) | MVP 제외, 필요 시 P1 검토 |

---

## 4. 사용자 스토리

- **팀원으로서**, 업무를 끝냈을 때 투두 체크박스를 클릭해 완료 처리하고 싶다. 그래야 내 대시보드에서 남은 일만 깔끔하게 볼 수 있기 때문이다.
- **팀원으로서**, AI가 추정한 마감일이 잘못됐을 때 직접 수정하고 싶다. 그래야 내 실제 일정에 맞게 할 일을 관리할 수 있기 때문이다.
- **팀원으로서**, 잘못 생성된 투두(내가 담당자가 아니거나 의미 없는 항목)를 삭제할 수 있길 원한다. 그래야 무관한 할 일로 인해 대시보드가 오염되지 않기 때문이다.
- **팀 리더로서**, 팀원이 자신의 투두를 완료 처리했는지 회의록 상세에서 확인하고 싶다. 그래야 후속 업무 진행 현황을 파악할 수 있기 때문이다.

## 5. 기능 요구사항 및 인수 기준

### 5.1 완료 처리

**Given/When/Then**

- Given 사용자가 홈 대시보드 또는 회의록 상세에서 자신의 투두 항목을 보고 있을 때
- When 체크박스를 클릭하면
- Then 해당 투두가 즉시 완료 상태로 변경되고, "진행 중" 목록에서 "완료 목록"으로 이동하며, 완료 처리 일시가 기록된다

**인수 기준**

- [ ] 완료 처리 시 체크박스가 ☑로 변경되고 투두 텍스트에 취소선 적용
- [ ] 완료된 투두는 **삭제되지 않고** "완료 목록"으로 이동 (이력 보존)
- [ ] 완료 처리는 **본인 투두만** 가능 (타인 투두 체크박스 비활성)
- [ ] 완료 처리 결과가 홈 대시보드와 회의록 상세 양쪽 모두에 동기화됨
- [ ] 완료 처리 후 별도 저장 버튼 없이 즉시 반영 (자동 저장)

### 5.2 마감일 수정

**Given/When/Then**

- Given 사용자가 자신의 투두 항목의 마감일 영역을 클릭하면
- When 날짜 피커(datepicker)가 인라인으로 표시되면
- Then 날짜를 선택하고 확인하면 마감일이 즉시 업데이트된다

**인수 기준**

- [ ] 마감일 클릭 시 인라인 날짜 피커 표시 (모달 이동 ✕)
- [ ] 오늘 이전 날짜도 선택 가능 (이미 지난 마감일 기록 허용)
- [ ] 날짜 피커 외부 클릭 시 변경 취소 (원래 날짜 유지)
- [ ] "마감일 미정"으로 되돌리는 옵션 제공 (날짜 피커 내 "미정으로 설정" 버튼)
- [ ] 수정된 마감일이 홈 대시보드 정렬 순서에 즉시 반영

### 5.3 투두 삭제

**Given/When/Then**

- Given 사용자가 자신의 투두 항목에 마우스 오버(또는 모바일 길게 터치)하면
- When [삭제] 아이콘(🗑️)이 나타나고 클릭하면
- Then 확인 모달이 표시되고, 확인 시 투두가 영구 삭제된다

**인수 기준**

- [ ] 삭제 전 확인 모달 필수: "이 할 일을 삭제할까요? 삭제 후 복구할 수 없습니다."
- [ ] 확인 모달 버튼: [삭제] (빨간색) / [취소]
- [ ] 삭제는 **본인 투두만** 가능
- [ ] 삭제된 투두는 해당 회의록 상세의 참여자별 할 일 섹션에서도 제거됨
- [ ] 삭제 후 대시보드에서 즉시 사라짐

### 5.4 투두 상태 표시 규칙

| 상태 | 시각적 표현 |
|---|---|
| 미완료 · 마감 여유 | 기본 텍스트, 회색 마감일 |
| 미완료 · 마감 3일 이내 | 기본 텍스트, 빨간색 마감일 |
| 미완료 · 마감 초과 | 기본 텍스트, 빨간색 마감일 + "⚠️ 지연" 배지 |
| 완료 | 취소선, 회색 텍스트, ☑ 체크박스 |

## 6. 기술 고려사항

- **낙관적 업데이트(Optimistic Update)**: 완료 처리 클릭 즉시 UI를 업데이트하고 백그라운드에서 API 호출, 실패 시 롤백
- **동시성**: 동일 투두를 여러 세션에서 접근하는 경우는 MVP에서 고려하지 않음 (단일 담당자 기준)
- **API 엔드포인트**:
  - `PATCH /todos/:id` — 완료 처리 (`is_completed: true`), 마감일 수정 (`due_date`)
  - `DELETE /todos/:id` — 삭제
